# Настройка приёмника UM980 для работы с B1C

## Проблема
Приёмник UM980 не видит сигналы B1C при отдельной генерации (без B1I).

## Решение

### 1. Проверка текущей конфигурации
Подключитесь к приёмнику и проверьте текущую группу сигналов:
```
CONFIG
```

### 2. Включение всех частот BeiDou
Для включения B1C и всех доступных частот используйте:
```
CONFIG SIGNALGROUP 2
SAVECONFIG
```

### 3. Поддерживаемые сигналы UM980
После настройки SIGNALGROUP 2 будут доступны:
- **BeiDou**: B1I, B2I, B3I, B1C, B2a, B2b
- **GPS**: L1C/A, L1C, L2C, L2P(Y), L5
- **GLONASS**: G1, G2, G3
- **Galileo**: E1, E5a, E5b, E6
- **QZSS**: L1C/A, L1C, L2C, L5

### 4. Особенности B1C в UM980
- UM980 имеет 1408 аппаратных каналов
- Холодный старт: <35 сек
- Тёплый старт: <10 сек
- Повторный захват: 1 сек

### 5. Почему B1C не определяется отдельно
Даже после настройки SIGNALGROUP 2, приёмник может использовать B1I-ассистированную стратегию захвата B1C по следующим причинам:
- Сложность BOC модуляции B1C
- Длинный период кода B1C (10 мс против 1 мс у B1I)
- Оптимизация времени захвата при холодном старте
- Встроенные алгоритмы Unicore NebulasIV

### 6. Рекомендации для тестирования
1. Убедитесь, что включен SIGNALGROUP 2
2. Для standalone B1C увеличьте время наблюдения (>5 минут)
3. Используйте сильный сигнал (открытое небо)
4. Для надёжного тестирования генерируйте B1I+B1C вместе

### 7. Команды для диагностики
```
GNGGA           # Проверить текущую позицию
GNGSV           # Список видимых спутников
CONFIG          # Текущая конфигурация
SIGNALGROUP     # Проверить группу сигналов
```

## Вывод
UM980 технически поддерживает B1C, но его алгоритмы оптимизированы для B1I-ассистированного захвата. Для тестирования B1C рекомендуется генерировать его вместе с B1I.
