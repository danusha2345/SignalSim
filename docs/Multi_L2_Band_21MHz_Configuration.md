# Мультисистемный пресет L2 диапазона с полосой 21 МГц

## Обзор конфигурации

Данный пресет предназначен для одновременной генерации сигналов нескольких GNSS систем в L2 диапазоне с полосой пропускания 21 МГц.

## Параметры конфигурации

### Основные параметры
- **Центральная частота**: 1217.37 МГц
- **Частота дискретизации**: 21 МГц
- **Формат данных**: IQ8 (8-бит I/Q)
- **Длительность**: 60 секунд

### Частотный план

С центром на 1217.37 МГц и полосой 21 МГц охватывается диапазон:
- **Нижняя граница**: 1206.87 МГц
- **Верхняя граница**: 1227.87 МГц

## Технические особенности

### Совместное использование частот
- **BeiDou B2I/B2b и Galileo E5b** используют одну и ту же несущую частоту (1207.14 МГц)
- Различаются по PRN кодам и навигационным сообщениям
- B2I и B2b от одного спутника BeiDou также разделяются по коду

### Промежуточные частоты (IF)
- GPS L2C/L2P: +10.23 МГц от центра
- BDS B2I/B2b и GAL E5b: -10.23 МГц от центра

### Модуляция
- **GPS L2C**: BPSK с TDM (чередование L2CM/L2CL)
- **GPS L2P**: BPSK (упрощенный P-код)
- **BeiDou B2I**: BPSK
- **BeiDou B2b**: BPSK  
- **Galileo E5b**: BPSK (I5 + Q5 в квадратуре)

### Навигационные сообщения
- **GPS L2C/L2P**: CNAV/LNAV соответственно
- **BeiDou B2I**: D1/D2 NAV
- **BeiDou B2b**: B-CNAV3
- **Galileo E5b**: I/NAV

## Требования к приемнику

### Минимальные требования
- Поддержка L2 диапазона (1200-1250 МГц)
- Полоса пропускания ≥ 21 МГц
- Многосистемная поддержка (GPS + BDS + GAL)

### Рекомендуемые приемники
- Septentrio PolaRx5S
- Trimble Alloy
- Javad TRE-3
- NovAtel PwrPak7
- u-blox ZED-F9P (ограниченная поддержка)

## Примечания по производительности

### Вычислительная нагрузка
- 5 различных типов сигналов одновременно
- Высокая частота дискретизации (21 МГц)
- Рекомендуется использование OpenMP для параллелизации

### Размер выходного файла
- Скорость генерации: ~42 МБ/с (21 МГц × 2 байта)
- Размер файла за 60 секунд: ~2.52 ГБ

## Известные ограничения

1. **ГЛОНАСС G2 не включен** - FDMA сигналы требуют специальной обработки
2. **Высокие требования к процессору** - рекомендуется многоядерная система
3. **Большой размер файла** - убедитесь в наличии свободного места

## Дополнительные настройки

### Увеличение мощности сигнала
Если приемник не захватывает сигналы, увеличьте мощность:
```json
"initPower": {
    "unit": "dBHz",
    "value": 50
}
```

### Уменьшение длительности
Для быстрого тестирования можно уменьшить время:
```json
"trajectoryList": [
    {
        "type": "Const",
        "time": 10.0
    }
]
```
